US Cellular CallDump System

Frequently Asked Questions

 Question  Answer
Queue
What does colors Yellow, Green and Red Represent?
 YELLOW means request is in progress.  GREEN indicates request is Ready.  PURPLE indicates request is on-hold and RED shows a request that failed.
How is the priority of a request raised?
 Ask the Call Dump admin to click the ↑ icon to raise the priority on the request.
How is the priority of a request lowered? Ask the Call Dump admin to click the ↓ icon to lower the priority on the request.
How is the status of a request changed?
 Ask the Call Dump admin to click the ∇ icon to change the status of the request. Status changes from Ready to Wait and Wait to Ready is possible on requests not currently being processed.
How is a request deleted?
 Ask the Call Dump admin to click the ∅ icon to remove the request. Requests in YELLOW are being processed and cannot be removed.
What happens when the request is completed.
 Calldump Reqeusts will be emailed to the email address on record for the user who submitted the request. The requestor will receive an separate report for each switch type (Nortel, Lucent) for each market.
Submitting Requests
How do I know I have choosen valid paramenters for the request?
 As the Request parameters are being entered, the TITLE bars above the individual section will turn RED or GREEN indicating that section has valid entries.  The rules for a valid request are:
  1. At least one search string must be entered.
  2. At least one switch must be entered
  3. Start date must be before the end date
  4. Start date cannot be more than 1 year old
  5. End date cannot be in the future
What happens if invalid parameters are choosen?
 If the request has been submitted, an "Invalid Request" page will be displayed with a message indicating the data that was incorrect. The request will have to be re-created.
User Administration
How is a new user added to the system?
 The designated ADMIN should build the user by clicking on the "Add User" link.
How is a user removed from the system?
 The designated ADMIN should click the "Change Status" link on the "Show Users" screen.
Troubleshooting
A request was submitted, appeared in the queue and disappeared without being emailed to the requestor. What happened? The following are possible causes:
  1. An invalid email addressed was entered when the user was created
  2. The report(s) are stuck in the mail queue and will arrive later
  3. A unknown technical problem has occurred.
Send email to the USCDL IS Back Office Development - MPS or USCDL EUS Back Office Support - MPS describing the problem.
