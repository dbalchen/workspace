Introduction

The Monitor is a system monitoring utility, which performs periodic audits and reports on the status of the MPS system activity.
The Monitor was written to enhance the ability of the MPS personnel to identify and respond to real and potential issues in a prompt manner thus minimizing customer impact.

Top

Architecture

The Monitor system composes of three tiers:
  • User Interface Tier
  • Collector Tier
  • Audit Tier

View the high level Monitor System Diagram.

Top

User Interface Tier

The User Interface Tier retireves information gathered in the Collector Tier and formats it appropriately for output.
Currently two interfaces exist.

The Web Monitor is a Java Servlet, which formats the plain text system status information received from the Collector Tier into HTML and allows access via a web browser.

The Dialer is a CRON-like Java application which sends a text page to an oncall associate if the collected output indicates a potentially threatening system condition.
The behavior of the Dialer is controlled through the monitor.xml configuration file. Please see the Audit Configuration Section of the User Manual for details on manipulating the Dialer behavior.

Additional interfaces may be added at any time following appropriate Change Management procedures.

Top

Collector Tier

The Collector Tier requests and gathers information from the Audit Tier at specified time intervals. Once the data is collected, the Audit Tier makes it available to the User Interface Tier servlet via a socket.

The collection is performed by daemon modules, which reside on each production machine. There is one Collector Daemon on for each market.

The Audit Section of the monitor.xml configuration file tells the Daemons how often to run a particular audit, the name of the executable to run, and the parameters to pass at the command line. The Collector executes the audit program, captures its STDOUT output, and stores it as the most up to date information regarding the particular Audit.

Top

Audit Tier

The Audit Tier is a collections of stand-alone programs which perfom individual system checks. As a rule, the programs are executed by the Collector Tier and report their findings to STDOUT, which is promptly collected by the Collector.=
